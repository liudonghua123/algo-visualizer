## 3. 核心组件设计

### 3.1 SortingVisualizer 组件
**职责**: 
- 管理整体应用状态
- 协调子组件通信
- 处理排序算法执行逻辑

**状态管理**:
```typescript
interface SortingState {
  array: number[];           // 当前数组
  sorting: boolean;           // 是否正在排序
  algorithm: string;          // 当前算法
  speed: number;              // 动画速度(ms)
  comparisons: number;        // 比较次数
  swaps: number;              // 交换次数
  highlightedIndices: number[]; // 高亮索引
  sortedIndices: number[];    // 已排序索引
}
```

### 3.2 BarChart 组件
**职责**:
- 渲染柱状图可视化
- 根据数组值渲染不同高度的柱子
- 应用颜色状态（比较/交换/已完成）

**Props**:
```typescript
interface BarChartProps {
  array: number[];
  highlightedIndices: number[];
  sortedIndices: number[];
  maxValue: number;
}
```

### 3.3 Controls 组件
**职责**:
- 算法选择按钮组
- 播放/暂停/重置控制
- 速度调节
- 数组大小滑块

### 3.4 StatsDisplay 组件
**职责**:
- 显示比较次数
- 显示交换次数
- 显示执行时间

## 4. 算法实现

### 4.1 生成器函数模式
所有排序算法使用 Generator 函数实现，支持逐步执行：

```typescript
function* bubbleSort(array: number[]): Generator<SortingStep> {
  const arr = [...array];
  const n = arr.length;
  
  for (let i = 0; i < n - 1; i++) {
    for (let j = 0; j < n - i - 1; j++) {
      yield { type: 'compare', indices: [j, j + 1] };
      
      if (arr[j] > arr[j + 1]) {
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
        yield { type: 'swap', indices: [j, j + 1], array: [...arr] };
      }
    }
    yield { type: 'sorted', index: n - i - 1 };
  }
  yield { type: 'sorted', index: 0 };
}
```

### 4.2 步骤类型定义
```typescript
interface SortingStep {
  type: 'compare' | 'swap' | 'sorted';
  indices?: number[];
  array?: number[];
}
```

## 5. 动画控制

### 5.1 速度控制
| 等级 | 延迟时间 | 适用场景 |
|------|----------|----------|
| 慢速 | 500ms | 教学演示 |
| 中速 | 200ms | 一般观察 |
| 快速 | 50ms | 快速验证 |

### 5.2 动画实现
- 使用 `requestAnimationFrame` 配合 `setTimeout` 控制执行节奏
- CSS `transition` 实现柱子高度和颜色变化
- 使用 React 状态更新触发重渲染

## 6. 状态管理流程

### 6.1 排序执行流程
```
用户点击播放
    ↓
创建 Generator 实例
    ↓
执行 next() 获取步骤
    ↓
根据步骤类型更新状态
    ↓
React 重新渲染
    ↓
等待延迟时间
    ↓
继续执行下一步骤
    ↓
直到排序完成
```

### 6.2 状态更新
- **比较状态**: 设置 `highlightedIndices`
- **交换状态**: 更新 `array` 和 `highlightedIndices`
- **完成状态**: 将索引加入 `sortedIndices`

## 7. 性能优化策略

### 7.1 渲染优化
- 使用 `React.memo` 优化子组件
- 避免不必要的重渲染
- 使用 CSS transform 而非修改位置属性

### 7.2 大量数据处理
- 限制数组大小在 10-100 之间
- 使用虚拟化技术（如需要更大规模）
- 批量状态更新减少重渲染次数
